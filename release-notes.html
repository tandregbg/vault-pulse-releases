Changelog

1.5.2 — 2026-04-27

Changed

  • Today / Recent now use modification time by default (CR-002). Previously these sections used filesystem creation time, which made files synced from another Mac (iCloud, Dropbox) appear missing -- the original creation date carries over from the source machine and never matches today on the receiving Mac. Modification time tracks "what did I touch today?" honestly and survives sync.
  • New Settings option ("Today / Recent uses:") with three values: Modified date (default), Created date (old behaviour), or Either. Switch under Cmd+, if the new default does not match how you think about your work.
  • Search panel result list also shows the chosen date, so what you sort by matches what you see.

Migration

  • Pre-1.5.2 configs without the new field auto-resolve to "Modified". To restore the old behaviour explicitly, open Settings and pick Created date.

1.5.1 — 2026-04-26

Added

  • "What's New" panel in the Sparkle update dialog. The full changelog (every version, newest first) is now rendered inside the dialog so users updating across multiple versions see everything they missed. Generated from CHANGELOG.md on every release, hosted as release-notes.html in the public companion repo.

1.5.0 — 2026-04-26

Added

  • Sparkle 2.x auto-updater wired into the app. New "Check for Updates..." menu item under the Settings entry. SPUStandardUpdaterController initialised at launch with a 24-hour automatic check interval.
  • release.sh: one-command release pipeline (bump → build → notarize → zip → Sparkle-sign → appcast → GitHub Release). Gated by DEVELOPER_ID + NOTARYTOOL_PROFILE env vars.
  • docs/release-process.md: one-time credentials setup walkthrough (Apple Developer cert, notarytool profile, Sparkle key, GitHub auth) and per-release operating instructions.
  • Public companion repo vault-pulse-releases hosts the appcast and signed zip downloads, so the source repo can stay private without breaking Sparkle's anonymous fetches.

Changed

  • build-app.sh now bundles Sparkle.framework + nested XPCs into Contents/Frameworks/, adds @executable_path/../Frameworks to the binary's rpath, and signs all nested binaries before the host app. Signing identity defaults to ad-hoc when DEVELOPER_ID is unset, switches to hardened-runtime + Developer ID when set, and additionally submits to Apple's notary service when NOTARYTOOL_PROFILE is also set.
  • First signed + notarized release. Recipients no longer need xattr -dr com.apple.quarantine; macOS Gatekeeper accepts the build directly.
  • README distribution section now distinguishes three flows: developer self-update (update.sh), signed/notarized release (release.sh), and quick unsigned share (package.sh).

1.4.0 — 2026-04-24

Added

  • First-run welcome window. When ~/.vault-pulse.json is absent, VaultPulse opens a welcome screen with:
  • Auto-detected Obsidian vaults (read from Obsidian's own obsidian.json registry, with a find-based fallback that also discovers un-opened vaults).
  • Badges on each row indicating Obsidian-registered vaults, vaults that match the core-skills layout (CLAUDE.md / _inbox/ / _tasks.yaml), and vaults that already have an _outbox/.
  • "Create a new vault" flow: pick parent folder, name it, optionally write a CLAUDE.md stub. Always creates _outbox/ for md2pdf / outbox-share compatibility.
  • "Use a custom folder" option for non-Obsidian layouts.
  • Footer link to the core-skills project.
  • New VaultDetector module (Obsidian registry reader + filesystem scan + real-path dedup). Also used to ensure _outbox/ exists for any selected vault.

Changed

  • Settings.shared gained isFirstRun and reload() to support the first-run flow without singleton re-initialisation.
  • Two scan windows instead of one: recentDays (default 7) drives the menu bar's "Recent" section; new searchDays (default 30) drives the search panel's default index. Deep search expands beyond both. Existing pre-1.4 configs auto-migrate -- searchDays falls back to max(recentDays, 30) when missing.
  • Search panel scope indicator (below the search field) is now a labelled button: "Last 30 days -- click to search all" makes the deep-search toggle discoverable instead of relying on the icon alone.
  • Menu bar "Recent" header now shows the active window, e.g. "Recent (7 days)".
  • Menu-bar menu gains "Welcome & vault picker..." so you can re-open the welcome screen and reconfigure the vault after first run.

1.3.5 — 2026-04-24

Changed

  • Search panel hint bar now fits on a single line -- uses \u{2318} / \u{21E7} / arrow symbols instead of spelled-out Cmd+ / Shift+ prefixes. The added Outbox hint in 1.3.4 had pushed the bar to wrap.

1.3.4 — 2026-04-24

Added

  • Cmd+Shift+P generates PDF into the vault's _outbox/ instead of the source folder (for mailing/sharing workflow). Cmd+P still writes next to the source file.
  • Startup environment check: resolves python3, timeout and verifies md2pdf's Python deps (markdown, weasyprint). If PDF generation would fail, posts a one-time notification explaining what to install.
  • PDF generation failures surface as a notification with the underlying error (previously failed silently).

Fixed

  • Cmd+P (PDF generation) no longer silently fails on Macs where python3 is only in /opt/homebrew/bin. GUI apps on macOS don't inherit the user's shell PATH, so VaultPulse now resolves python3 explicitly via a small PATH probe before shelling out to md2pdf.
  • md2pdf now invoked with explicit -o <dir> so output path is deterministic regardless of CWD.

1.3.3 — 2026-04-24

Fixed

  • Scanner no longer silently fails on Macs without GNU timeout installed -- SpotlightScanner and FileScanner now resolve timeout/gtimeout at runtime and fall back to bare find when neither is present. Previously, a missing timeout binary (default on macOS) caused bash -c to error out before find ran, producing 0 results with no visible diagnostic.

1.3.2 — 2026-04-24

Fixed

  • SSHFS remotes scanner now scans mount root instead of mount/src/ (mounts changed to expose content directly)
  • Fixed find -not -path '/.' breaking SSHFS deep directory traversal -- replaced with -not -name '.*'
  • Fixed find -type f returning no results on SSHFS during deep traversal -- removed type filter, uses grep for extension matching
  • Removed per-file stat calls on SSHFS remotes (caused hangs on some files) -- uses filename date parsing instead
  • Undated SSHFS files no longer appear as "created today" -- assigned distantPast instead of now()
  • Added 2-second liveness check before scanning each SSHFS mount -- frozen mounts are skipped instead of blocking
  • Stuck SSHFS find processes now killed with SIGKILL (SIGTERM cannot interrupt uninterruptible sleep)
  • Partial scan results preserved when mount scan times out (writes to temp file instead of pipe)
  • Increased per-mount scan timeout to 30s and remotes group timeout to 35s for deep directory trees
  • Added app icon: vault door with document on blue-teal gradient
  • Build script now generates AppIcon.icns from source PNG automatically

1.3.1 — 2026-04-22

Fixed

  • Vault scanner rewritten from Spotlight (mdfind) to shell find + stat -- finds all iCloud files including cloud-only files not yet in local Spotlight index
  • No more TCC "network volume access" prompt at launch -- removed FSEvents watcher on iCloud path, deferred SSHFS remotes scan to second refresh cycle
  • Fixed mdfind pipe buffer deadlock that caused 20s timeout on large vaults (500+ files)
  • Fixed mdfind -attr output parser that leaked attribute text into filenames
  • Scan results cache persisted to disk for instant startup with stale data while fresh scan runs
  • Extension filter chips in search panel (md, pdf, txt, doc, xls, ppt)
  • Cmd+F reveals selected file in Finder

1.3.0 — 2026-04-21

Added

  • SSHFS remotes support: scans src/*/ inside each mount at ~/workspace/remotes/
  • Parallel remote scanning with 15-second timeout per mount
  • SSHFS file opening: copies to temp via shell glob to work around macOS NFC/NFD mismatch
  • Whitelist-based file filtering: only show document types (md, txt, pdf, docx, pptx, xlsx, etc.)
  • Two-phase scanning: vault results appear instantly, remotes follow when ready
  • Symlinked directories skipped to avoid double-scanning vault

Changed

  • Replaced blacklist (excludePatterns) with whitelist (includeExtensions) + directory exclusions (excludeDirs)
  • Default extensions: md, txt, pdf, doc/docx, xls/xlsx, ppt/pptx, csv, odt, ods, odp, key
  • No images by default (add via Settings if needed)
  • Full filenames shown everywhere (no more date prefix stripping)
  • Settings UI updated with file types whitelist and remotes path fields
  • Scan guard prevents overlapping scans from piling up

Fixed

  • SSHFS files with Swedish characters (å, ä, ö) can now be opened in Typora
  • Workspace scan no longer follows symlinks into the vault (caused hangs)
  • Remote mount scan no longer blocks the entire app
